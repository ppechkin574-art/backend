# AIMA — Продуктовый Roadmap

Документ описывает **продуктовые** улучшения (UI/UX, контент, локализация, фичи) — то что увеличивает ценность приложения для конечного пользователя и конверсию из free → paid. Технический долг (инфра, безопасность, рефакторинг) — в [TECH_DEBT.md](TECH_DEBT.md), не дублируется здесь.

**Создан:** 16.05.2026
**Стейкхолдер:** оператор (single decision-maker)
**Формат:** приоритезированный backlog без жёстких сроков. Берём верхнюю задачу когда есть время / команда / бюджет.

---

## Как читать оценки

### Impact — влияние на продукт + конверсию

- 🔥 **HIGH** — задача напрямую двигает юзера free → paid, ИЛИ предотвращает churn у paid. Деньги.
- 📈 **MEDIUM** — повышает воспринимаемую ценность, NPS, retention. Деньги — но через несколько шагов.
- ✨ **LOW** — гигиена / косметика. Не двигает метрики напрямую, но без неё продукт ощущается дешёвым.

### Effort — оценка трудозатрат

- **S** (small) — несколько часов на одного разработчика
- **M** (medium) — 1-5 рабочих дней
- **L** (large) — 1-3 недели
- **XL** — месяц+ (часто включает контентную работу или внешних людей: дизайнер, преподаватели)

Effort оценивается **консервативно** — реальное время обычно × 1.3-1.5 из-за тестирования, ревью, доработок.

## 🎨 UI / UX

### Полировка существующих экранов

#### **U-1. Сделать subscription screen storytelling, а не списком фич** [🔥 HIGH] [S — 4-6ч дизайн + 1д код]

Сейчас экран подписки = таблица галочек по фичам. Это слабый convert-копирайт. Современный pattern (Notion, Duolingo, Headspace): экран рассказывает историю — «Готовься к ЕНТ уверенно: персональный план, 5000+ вопросов, разбор каждой ошибки. Месяц всего 4990 ₸». Большая кнопка, контр-аргументы внизу маленьким (cancel anytime, no auto-charge…). Это **дешёвая правка с прямым влиянием на конверсию**.

#### **U-2. Empty states + skeleton-загрузка** [📈 MEDIUM] [M — 2-3д]

Сейчас при пустых данных юзер видит белый экран. Везде. Везде, где грузится список (тренажёр, лидерборд, профиль) — нужен skeleton-плейсхолдер вместо спиннера; при пустом результате — иллюстрация + CTA («Ты ещё не решал тесты — начни прямо сейчас»). UX-стандарт 2026, без этого приложение ощущается сырым.

#### **U-3. Speed: загрузка экранов 5-7 секунд** [🔥 HIGH] [M — 2д Flutter + 1д Cloudflare]

Уже в TECH_DEBT.md (#8). Тут дублирую как продуктовую задачу — это **самый частый рейтинговый отзыв «приложение тормозит»**. Flutter TTL-кэш для plans/benefits/user (2-3ч TTL) + Cloudflare proxy перед Railway. Без этого все остальные UX-улучшения нивелируются «всё равно тормозит».

#### **U-4. Auto-logout каждые 30 минут** [📈 MEDIUM] [S — 30 минут]

TECH_DEBT.md #9. Юзер открывает приложение → попадает на экран входа → теряет интерес. Фикс на 30 минут в Keycloak (увеличить refresh-token TTL до 30 дней) + 15 минут в Flutter (`iOptions` в `flutter_secure_storage`). Тривиально, но влияние на retention существенное.

#### **U-5. Анимации правильный/неправильный + микро-celebration на серии** [📈 MEDIUM] [M — 2-3д]

После ответа на вопрос — короткая (200-300мс) анимация: зелёный чек с лёгким bounce, или красный X с shake. На каждой 5-й правильной подряд — салют/конфетти на 1 сек. Это создаёт **эмоциональную привязку** — Duolingo на этом построил retention.

#### **U-6. Тёмная тема** [📈 MEDIUM] [M — 3-4д]

ENT-студенты учатся вечером и ночью. Светлая тема жжёт глаза. Дополнительная переменная брендинга (purple на тёмном фоне выглядит дорого). Технически — это переписать все цвета через `Theme.of(context).colorScheme` где этого ещё нет.

#### **U-7. Версия в футере профиля показывает "01.01.01"** [✨ LOW] [S — 30 минут]

TECH_DEBT мелочь, но визуально торчит. Правильно показывать `1.2.0 (22)` из `pubspec.yaml` + build number.

#### **U-8. Onboarding для первого запуска** [🔥 HIGH] [M — 3-4д]

Сейчас первый запуск = welcome screen → регистрация → пустой профиль. Нужен 3-4 экранный onboarding: «1) Прохожу диагностику (3 вопроса) 2) Получаю прогноз моего балла 3) Получаю план до ЕНТ». Это **первая точка конверсии** — за этот flow юзер либо «зацепился» либо ушёл. Связано с задачей **B-8** (персональный план).

#### **U-9. Pull-to-refresh + правильные error states с retry** [📈 MEDIUM] [S — 1д]

Везде где список — добавить pull-to-refresh; везде где запрос — на ошибку показать «Что-то пошло не так. Попробовать ещё раз» с кнопкой retry. Сейчас при сетевом сбое юзер видит вечный спиннер или белый экран.

#### **U-10. Поиск по темам/вопросам** [📈 MEDIUM] [M — 2-3д]

Глобальный search bar — ввёл «синусы» → нашёл темы из Математики, релевантные вопросы. Закрывает кейс «помню что было что-то про синусы — где найти». Backend: `/search?q=…`, fuzzy match по `Topic.name + Question.body`.

#### **U-11. Bookmark / избранные вопросы** [📈 MEDIUM] [M — 2д backend + 2д Flutter]

«⭐ В избранное» на экране вопроса. Список избранных в профиле. PRO: безлимит избранных; free: лимит 10. Закрывает кейс «хочу пересмотреть сложные вопросы потом» + создаёт мини-paywall.

#### **U-12. Push-уведомления — настройки в профиле** [📈 MEDIUM] [S — 1д]

Сейчас юзер либо получает все пуши либо никакие (системная настройка iOS). Нужны категории внутри приложения: ежедневный тест, новости, акции, родительский отчёт. Без этого юзер отключает ВСЕ пуши при первом раздражении — теряем канал.

### Новые UX-фичи

#### **U-13. Visualisation прогресса по предметам** [📈 MEDIUM] [M — 3-4д]

На главном экране — «карточки предметов» с прогресс-баром «решено 234 / 500 вопросов» и средним баллом по теме. Это **визуальный мотиватор**: юзер видит «История 12%» и хочет «закрыть» предмет. Также **anchor для конверсии** — рядом с прогресс-баром «PRO открывает доступ к ещё 1200 вопросам».

#### **U-14. Achievements / бейджи** [📈 MEDIUM] [L — 1-2 недели]

Связан с гемификацией (B-6). Бейджи за вехи: «Первый идеальный тест», «10 дней подряд», «Закрыл предмет», «500 правильных подряд». Видны в профиле. Создают **collectibles loop** — мощный retention-механизм, free-юзеры стремятся «собрать всё», увеличивают time-in-app.

#### **U-15. Review mistakes flow** [🔥 HIGH] [M — 3д]

После теста — отдельный экран «Твои ошибки (4)»: каждая ошибка показывается заново с правильным ответом и пояснением. Можно «пройти ошибки ещё раз». Это **главный обучающий механизм** — школьник учится из ошибок, а сейчас они просто исчезают из таблицы попыток.

#### **U-16. Daily challenge виджет на главной** [📈 MEDIUM] [M — 2д]

«Сегодняшний челлендж: 5 вопросов из истории Казахстана за 3 минуты». Свежее каждый день, генерится backend'ом. Free-юзер получает бесплатный челлендж в день, PRO — безлимит + retry. Создаёт **причину открыть приложение каждый день** + конверсия через «попробуй ещё раз — только в PRO».

## 📚 База вопросов

### Q-1. Количество — добраться до 500+ вопросов на главные предметы [📈 MEDIUM] [XL — контентная]

Аудит по предметам: сколько сейчас вопросов в каждом, где «тонко». Цель — минимум 500 вопросов на каждый из обязательных ЕНТ-предметов (Математика, История Казахстана, Грамотность), минимум 300 на каждый профильный. **Контентная работа** — нужны преподаватели/методисты, не разработчики. Создаёт фундамент: без массы вопросов всё остальное бессмысленно.

### Q-2. Качество — пояснение «почему» к каждому вопросу [🔥 HIGH] [XL — контентная + S код]

**Главное продуктовое отличие платного приложения от бесплатного YouTube**. Сейчас правильный ответ показывается как «А». Должно быть: «А правильно, потому что [объяснение]. Б неправильно, потому что [почему ловушка]. С/D — посмотри тему «такая-то».» Это превращает приложение из тренажёра в учителя. Backend готов (поле `explanation` в `Question`), нужна **контентная работа** — пройти по существующим вопросам и заполнить. Технически прикрутить отображение — несколько часов.

### Q-3. Качество — audit на кривые ответы [📈 MEDIUM] [L — контентная]

Аудит существующей базы: где правильный ответ помечен неверно (баги в исходных файлах), где формулировка двусмысленная, где варианты ответов нелогичны. Без community-report mechanism это пока ручная работа. Можно ускорить: показать в админке топ-100 вопросов с самым низким success rate (явный признак что что-то не так с вопросом) и пройтись по ним.

### Q-4. Качество — система подсказок (tier 1 → tier 2 → ответ) [📈 MEDIUM] [M — 2-3д код + контент]

Кнопка «подсказка» на вопросе:
- Tier 1: «эта тема — Геометрия, формула площади круга» (бесплатно)
- Tier 2: «подставь радиус в формулу π·r²» (PRO)
- Tier 3: показать ответ (PRO)

Делает обучение постепенным, не «угадал/не угадал». Free-юзер видит paywall **в момент когда хочет помощь** — лучшая точка конверсии.

### Q-5. Качество — заменить text-heavy вопросы на изображения / схемы [📈 MEDIUM] [L — контентная + S код]

Сейчас многие вопросы по геометрии / физике / биологии — текстовые описания где должна быть схема. Особенно влияет на сложные темы. UI готов (`Question.image_url`), нужна работа методистов.

### Q-6. Лучший tagging — difficulty levels + topics [📈 MEDIUM] [M — 2-3д код + L контент]

Сейчас вопросы тегируются по теме грубо. Нужна:
- Сложность (1-5 звёзд)
- Связь с конкретной подтемой из учебной программы
- Год появления в реальном ЕНТ (если использовался)

Позволит делать **адаптивный движок** (B-8 ниже) — без правильного tagging алгоритм слепой.

### Q-7. Новый формат — соответствия (match pairs) [📈 MEDIUM] [M — 3-4д]

Формат реального ЕНТ: «Сопоставь правителя с эпохой». Сейчас формат — только single-select из 4 вариантов. Соответствия — отдельный widget, отдельный scoring. **Покрытие реального формата ЕНТ** — без этого приложение готовит «к похожему», а не «к экзамену».

### Q-8. Новый формат — задачи с несколькими шагами (multi-step) [📈 MEDIUM] [M — 4-5д]

Для математики, физики, химии — задачи где надо ввести промежуточный результат, не только финальный ответ. Имитация письменной части ЕНТ. Гораздо более «учебно», чем single-select.

### Q-9. Новый формат — read-comprehension блоки для грамотности [📈 MEDIUM] [M — 3д]

ЕНТ-блок Чтение Грамотности — это текст + 5 вопросов по нему. Сейчас приложение их не покрывает. Это **критичный gap** для тех кто хочет реально готовиться.

### Q-10. Новый формат — audio-вопросы для иностранного языка [✨ LOW] [L — code + content]

Listening section на ЕНТ если юзер сдаёт English. Низкий приоритет — иностранный язык не для всех, и контентная работа большая (запись).

## 🌐 Локализация

### L-1. Вынести все UI-строки в i18n bundles [📈 MEDIUM] [M — 3-4д]

Сейчас тексты в коде Flutter жёстко зашиты на русском. Нужно: настроить `flutter_localizations`, вынести все строки в `app_ru.arb` и `app_kk.arb`. Это **обязательная подготовка** перед любой локализацией — без этого Kazakh-перевод невозможен.

### L-2. Перевод UI на казахский [🔥 HIGH] [M — 3-5д на перевод]

После L-1: пройтись по `app_kk.arb`, заполнить. Нужен носитель языка для нормального перевода (не Google Translate). Открывает рынок **kaz-говорящих школьников ≈30-40% сегмента ЕНТ**. Без казахского интерфейса мы теряем половину рынка, как бы крутой ни был контент.

### L-3. База вопросов на казахском [🔥 HIGH] [XL — контентная]

ЕНТ сдаётся на двух языках, школьник выбирает язык экзамена. Сейчас наши вопросы только на русском → kaz-школьник в нашем приложении видит русские формулировки → испытывает другой когнитивный путь чем на реальном экзамене → меньше пользы. **Огромная контентная работа** — перевести 3000+ существующих вопросов. Делаем по предметам по приоритету: сначала Математика + Грамотность Казахский Язык (там казахский — основа), потом профильные. Часто проще набирать новые вопросы сразу двуязычными.

### L-4. Auto-detect языка по системе + manual switch в настройках [✨ LOW] [S — 4ч]

При первом запуске берём `Platform.localeName` и подставляем. В Settings — toggle «Язык интерфейса». Тривиально, но без этого juzer видит русский по умолчанию даже если у него iPhone на казахском.

### L-5. Локализация admin-панели [✨ LOW] [M — 2-3д]

Низкий приоритет — админка для нас, не для конечных юзеров. Может пригодиться если придёт kaz-говорящий контент-менеджер.

## 🚀 Большие фичи (новые продуктовые направления)

Каждая из этих фич — самостоятельный продукт внутри AIMA. Эффект — кратный, но и работы кратной.

### **B-1. Батлы между участниками (PvP)** [🔥 HIGH] [L — 2-3 недели]

Реал-тайм-матч 1-vs-1: оба юзера получают одинаковый набор из 10 вопросов, побеждает тот кто быстрее правильно ответил. После — лидерборд по wins/losses. PRO: безлимит батлов; free: 3 батла в день.

**Почему это работает:**
- **Социальный hook** — школьники зовут одноклассников «давай зарубимся в AIMA»
- **Mini-paywall** — после 3 батлов «хочешь продолжить — PRO»
- **Time-in-app** — батлы короткие и addictive, юзер играет по 30 минут вместо плановых 10

**Технически:** WebSocket-pair-matchmaking (у нас уже есть WS для платежей, инфраструктура поднята), новая таблица `battles`, новые UI-экраны (lobby, in-match, result). Сложность средняя, риск (concurrency-баги) — средний.

### **B-2. AI-ассистент / помощник в обучении** [🔥 HIGH] [L — 2-3 недели на MVP, XL на полноценный]

Чат-бот внутри приложения. Кнопка «Помощник» на каждом вопросе → открывается чат: «Объясни эту тему», «Покажи похожий пример», «Я не понимаю шаг 3». LLM (Claude / GPT-4) с тонкой настройкой на ЕНТ-контент.

**Почему это работает:**
- **Voiced главное продуктовое отличие** — «У нас в приложении ИИ-учитель»
- **Прямая convert-история** — free видит «Спроси AIMA», нажимает, видит paywall «10 вопросов в день в PRO». В PRO — безлимит
- **Стоит копейки на технической стороне** — Claude/GPT-4 API ~$0.01-0.05 на сложный вопрос

**MVP-стратегия:** базовая интеграция с Claude API + системный промпт «ты учитель ЕНТ-предметов, отвечай по-учебному», лимиты для free (10 запросов в день), безлимит для PRO. Без галлюцинаций надёжно работать будет не на каждом вопросе — но даже 70% качества создаст огромный wow-эффект.

**Полноценный вариант:** RAG поверх нашей базы вопросов + методичек, fine-tuned модель на ЕНТ-контенте. Это уже месяцы работы.

### **B-3. Живые учителя — консультации внутри приложения** [📈 MEDIUM] [XL]

«Записаться на консультацию» → выбор предмета и времени → видео-звонок с преподавателем. Преподавателей либо нанимаем, либо marketplace-модель (преподаватель регистрируется, ставит цену, мы берём комиссию).

**Почему это работает:**
- **Premium-tier** — выше PRO. Условный «PRO+ с 4 консультациями в месяц за 19990 ₸»
- **Доверие** — «у вас есть живые учителя» — мощный аргумент при покупке
- **High-touch retention** — школьники, у которых был контакт с конкретным преподавателем, не уходят

**Сложности:**
- Нужны сами преподаватели — это операционка, не код. Найти, договориться, отрегулировать качество.
- Видеосвязь — Zoom embed / Daily.co API / Agora.io. Не код, не контент — но инфра-выбор и интеграция.
- Расписание/календарь — большая UX-задача.
- Юридически — мы становимся образовательной платформой с услугами, могут понадобиться разрешения.

**Рекомендация:** не делать сейчас. Сначала закрепить PRO-tier (B-1, B-2, B-6) — там операционка проще. Live-учителя — это **отдельный продукт-направление**, требует команду.

### **B-4. Расширенный лидерборд — школы + регионы** [📈 MEDIUM] [M — 4-5д]

Сейчас лидерборд — топ юзеров. Нужны:
- Лидерборд **по школам** (агрегат баллов всех юзеров школы)
- Лидерборд **по городам/областям** (агрегат всех школ)
- Personal — где я в моей школе, в моём городе, в Казахстане

**Почему это работает:**
- **Школьный hook** — школьник зовёт одноклассников «давайте подтянем нашу школу в топ»
- **PR / маркетинг** — «Школа №X — №1 в Алматы по AIMA» — отличный пост в соцсети школы
- **Конкуренция** — мотиватор который не зависит от наших усилий, работает сам

**Технически:** нужно собирать `school_id` при регистрации (сейчас не собираем) — добавить в onboarding (опциональное поле). Backend: новые агрегатные эндпоинты, скорее всего материализованные view в Postgres для скорости.

### **B-5. Родительский / семейный аккаунт** [🔥 HIGH] [L — 2 недели]

У нас уже есть Family feature от Романа (модель `quiz.services.family` существует). Но недо-эксплуатирована. Нужно:
- Полноценный родитель-аккаунт: подписывает PRO для всей семьи (до 4 детей)
- Дешборд родителя: «Алибек учился 2.3 часа на этой неделе, средний балл 75/100, слабее всего — Физика, последний раз учился 3 дня назад»
- Push-алерты родителю: «Сын не открывал приложение 5 дней» / «Дочь прошла полный тренажёр по Истории Казахстана»
- Цена family plan чуть выше single PRO (например 7990 вместо 4990), но дешевле чем 2× PRO

**Почему это работает:**
- **Платит родитель, не ребёнок** — родитель более платёжеспособен и более склонен к подписке «ради образования»
- **Конкретный benefit** — родители платят за прозрачность, школьники за подготовку
- **Higher ARPU** — family plan = 1.6× single, при затратах на привлечение тех же
- **Sticky** — родитель не отменит подписку «из-за того что у меня нет настроения», как мог бы школьник

### **B-6. Гемификация — коины, обмен на скидку/подарки** [🔥 HIGH] [L — 2 недели]

Юзер зарабатывает «коины» за активности: 5 за правильный ответ, 20 за тест 100%, 50 за полный пройденный topic, и т.д. Тратит на:
- **Скидка на подписку** — 5000 коинов = 10% скидка на месяц (механика «купи за реальные деньги дешевле, если активно учился»)
- **Косметика** — рамки на профиль, цветные темы, стикеры в чатах
- **Физические подарки** — топ-100 коинов в сезон получают мерч / стипендию / iPad
- **Внутренние перки** — больше hint'ов, доступ к сложным режимам, AI-помощник со скидкой

**Почему это работает:**
- **Daily-active-driver** — есть причина открыть приложение каждый день
- **Конверсия мягче** — «8000 коинов до бесплатной недели PRO» — это **накопительная мотивация**, юзер не уйдёт даже когда устал
- **Cashback-pattern** — мы уже умеем «возвращать на подписку» (есть FeatureType.CASHBACK), это его развитие
- **Виральность через подарки** — топ-100 в сезон, инстаграм-пост «я получила iPad от AIMA», свежая аудитория

**Сложности:**
- Экономика: сколько коинов за что давать, чтобы не было инфляции. Нужно симулировать на бумаге.
- Анти-чит (юзеры будут пытаться накручивать): server-side validation каждой выдачи коинов, не доверять клиенту.
- Backend: новые таблицы `user_coins`, `coin_events`, `rewards_catalog`.

### **B-7. Привести-друга — мутуальная неделя PRO** [🔥 HIGH] [M — 5-7д]

Реферальная программа. Юзер шарит ссылку через любой канал → друг скачивает по ней → регистрируется → подтверждает SMS → **оба получают неделю PRO**.

**Почему это работает:**
- **Двустороннее вознаграждение** — современный pattern (Dropbox, Notion, Cash App). Работает в разы лучше односторонних "ты получаешь, друг просто скачивает"
- **CAC = 0** — каждый привлечённый юзер стоит нам только неделю PRO (~80% маржа на этом). Это в 10× дешевле любой рекламы
- **Возрастная виральность** — школьники общаются массово, ссылки расходятся быстро
- **Free→Paid funnel** — за неделю PRO юзер привыкает к фичам, не хочет терять
- **Виральный k-фактор** — каждый PRO-юзер приводит 1-2 friends в месяц = устойчивый органический рост

**Технически:** генерация уникальной ссылки на юзера (`/r/<short_code>` → редирект на App Store/Play Store + сохранение reference в URL/iCloud paste). Backend: таблица `referrals`, эндпоинт «привязать referral после регистрации», cron на выдачу недели обоим (after-7-days-active чтобы избежать накруток). Flutter: экран «Пригласи друга», deep-link integration.

### **B-8. Персональный план до ЕНТ + countdown** [🔥 HIGH] [L — 2-3 недели]

При регистрации спрашиваем: «Когда ты сдаёшь ЕНТ?» + «Какие предметы?». Алгоритм:
- Видит твою дату ЕНТ (или дефолт — следующий май)
- Видит твои текущие результаты по предметам (постепенно набирает данные)
- Видит сколько вопросов и тем осталось «не закрытыми»
- Выдаёт **рекомендованный план на каждый день**: «Сегодня: 12 вопросов по математике (твоя слабая тема), 1 тест по Истории Казахстана для повторения, 5 минут на ошибки прошлой недели»
- На главном экране — countdown: «До ЕНТ осталось 89 дней. Готовность: 67%. Текущий прогноз балла: 102/140»

**Почему это работает:**
- **Главный value-driver** — школьник видит конкретный путь к результату, не абстрактные «учись с AIMA»
- **Daily reason to open** — план обновляется каждый день
- **Прямая convert-история** — free видит «Твой персональный план», нажимает — «Доступно только в PRO». Это **самый сильный convert-момент**: юзер уже понимает что план был сгенерирован под него, не хочет его терять
- **Прогноз балла** — самый ёмкий механизм tracking прогресса. «Мой прогноз вырос с 78 до 105 за месяц» — это шерится в соцсетях, продаёт за нас

**Сложности:**
- Алгоритм прогноза — нужна нормальная статистика по теме (Q-6 — tagging) + калибровка против реальных ЕНТ-результатов
- UX персонального плана — большая работа дизайна

### **B-9. Видео-объяснения для топ-100 сложных вопросов** [📈 MEDIUM] [XL — контентная + S код]

Для **самых сложных** вопросов (определяем по success rate) — 30-60-секундный видео-разбор от преподавателя «вот как это решается». Видео хранится в MinIO, у вопроса поле `explanation_video_url`.

**Почему это работает:**
- **Воспринимаемая ценность подписки** — «У нас 100+ видео-разборов» — конкретное число в маркетинге
- **Differentiation vs YouTube** — на YouTube надо искать самому, в AIMA — встроено прямо в нужный момент
- **Конверсия через paywall** — free видит иконку 🎥 на вопросе, нажимает — «Только в PRO»

**Сложности:**
- Контентная работа — нужен преподаватель + запись + монтаж. ~$30-50 на видео, на топ-100 = $3-5k единовременно. Окупается с 5-10 подписками.
- Стриминг видео из MinIO — нужна правильная настройка HTTP-streaming, ProgressiveDownload в Flutter. Технически решаемо, но обкатать надо.
