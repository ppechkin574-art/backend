## Задача 1 — Казахский язык в контенте (тесты, вопросы, темы)

### Анализ кода

**Что уже есть:**
- Кнопка выбора языка на экране входа (`language_selector_widget.dart`) — отображает "Рус" / "Каз", **но не делает ничего** (хранит строку в локальном стейте, не влияет ни на что)
- В моделях вопросов (`TrainingQuestionModel`, `Question`) — поля `lang` / `language` **нет**
- В базе данных (`quiz/models.py`) — таблицы `questions`, `topics`, `subjects` — поля `lang` **нет**
- API не принимает параметр языка ни в одном quiz-эндпоинте

**Вывод:** Нет ни одного слоя где язык обрабатывается. Задача требует изменений во всех 4 слоях: БД → API → Flutter-модели → UI.

---

### Что нужно сделать

#### Backend
- [ ] `B1.1` Добавить колонку `lang` (тип `VARCHAR(2)`, значения `ru`/`kk`, default `ru`) в таблицы: `questions`, `topics`, `subjects`
- [ ] `B1.2` Написать Alembic-миграцию
- [ ] `B1.3` Добавить query-параметр `?lang=ru|kk` в эндпоинты: `/quiz/topics`, `/quiz/questions`, `/training/...`, `/ent/...`, `/daily-test/...`
- [ ] `B1.4` Фильтрация: если контент на запрошенном языке отсутствует — fallback на `ru`
- [ ] `B1.5` Админ-панель: поле выбора языка при создании/редактировании вопроса

#### Flutter (оба разработчика)
- [ ] `F1.1` Сохранять выбранный язык юзера в `SecureStorage` / `ObjectBox` при регистрации и в настройках
- [ ] `F1.2` Передавать `lang` параметр во все API-запросы quiz-слоя
- [ ] `F1.3` Добавить `lang` в модели: `TrainingQuestionModel`, `Question`, `TopicModel`, `SubjectModel`
- [ ] `F1.4` Экран настроек: переключатель языка контента (отдельно от языка интерфейса)
- [ ] `F1.5` Language selector на экране входа — подключить к реальному стейту (сейчас заглушка)

## Задача 2 — Казахский язык интерфейса

### Анализ кода

**Что уже есть:**
- `flutter_localization: ^0.3.3` — пакет подключён
- `l10n.yaml` — конфиг для генерации локализации
- `assets/l10n/app_ru.arb` — русские строки (сгенерированы)
- `lib/core/presentation/src/l10n/` — `AppLocalizations`, `AppLocalizationsRu`, extension
- В `main.dart:59` — `const Locale('ru')` хардкод, казахский `Locale('kk')` **не добавлен**

**Вывод:** Инфраструктура локализации готова на 60%. Нужно добавить `app_kk.arb` с переводами и подключить переключатель.

---

### Что нужно сделать

#### Flutter (оба разработчика)
- [ ] `F2.1` Создать `assets/l10n/app_kk.arb` — перевод всех строк из `app_ru.arb` на казахский
- [ ] `F2.2` Добавить `AppLocalizationsKk` класс (по аналогии с `AppLocalizationsRu`)
- [ ] `F2.3` В `main.dart` — добавить `Locale('kk')` в `supportedLocales`, убрать хардкод
- [ ] `F2.4` Language selector (экран входа + настройки) — подключить к `flutter_localization` или `Locale` в `MaterialApp`
- [ ] `F2.5` Сохранять выбранный язык UI в `SecureStorage`, восстанавливать при запуске
- [ ] `F2.6` Проверить все экраны — нет ли хардкода русских строк вне `.arb` файла

---

### Вопросы перед стартом

1. **Сколько строк в `app_ru.arb`?** Нужно понять объём перевода. Если > 200 строк — стоит отдать переводчику, не делать самим.

2. **Казахский: кириллица или латиница?** Казахстан переходит на латиницу. Ученики 10–11 класса сейчас используют кириллицу (`қ`, `ң`, `ү` и т.д.). Какой вариант?

3. **Язык по умолчанию при установке?** Автоопределять по системной локали телефона (`kk_KZ` → казахский, иначе → русский)? Или всегда спрашивать при первом запуске?

4. **Есть ли уже готовые переводы?** У заказчика есть казахскоязычный контент-менеджер или придётся делать всё с нуля?
